# 입문자 안전 게이트 및 유형별 필수 조건 검사

이번 패치는 입문자가 키워드만 보고 공식을 찍는 상황을 줄이기 위해 추가되었습니다. 앱은 이제 `경사면`, `원운동`, `rolling`, `curve` 같은 단어만으로 구체식을 바로 제시하지 않습니다.

## 핵심 원칙

- 정보가 부족하면 적용식 영역을 비웁니다.
- 대신 후보 유형, 확인해야 할 조건, 쓰면 안 되는 식을 표시합니다.
- 일반 원리 수준의 안내는 허용하지만, 조건이 확정되지 않은 방정식은 적용식으로 제시하지 않습니다.
- 입문자 모드에서는 중복 경고를 줄이고 한 단계씩 확인하도록 안내합니다.

## 추가된 정보 부족 게이트

| 후보 유형 | 부족하면 멈추는 조건 | 보류되는 대표식 |
|---|---|---|
| 경사면 | 경사각, 마찰 조건, 운동 방향 부족 | `mg sinθ = ma`, `f = μN` |
| 원운동 장력 | 수평/수직 원운동, 최고점/최저점, 줄/트랙 모델 불명 | `T - mg = mv²/R`, `T + mg = mv²/R` |
| 구름 운동 | no slip 또는 slipping 조건 불명 | `v_G = ωR`, `a_G = αR` |
| 커브 주행 | 평평한 커브/경사진 커브 구분, 마찰/반지름 조건 부족 | `v_max = √(μ_s g R)`, `tanθ = v²/(gR)` |

## 질문형 그림 입력 마법사

`question_wizard_for_text()`는 감지된 후보 유형에 따라 추가 질문을 제공합니다.

예를 들어 경사면 후보에서는 다음을 묻습니다.

1. 경사각 theta가 주어졌나요?
2. 마찰이 없나요, 있나요? 있다면 μ_s 또는 μ_k가 주어졌나요?
3. 물체가 위로 움직이나요, 아래로 움직이나요, 아니면 정지 직전인가요?
4. 외력이 작용하나요?
5. 구하려는 값은 무엇인가요?

## 오답노트 자동 태그

저장 시 사용자가 틀린 이유를 직접 고르지 않아도, 비적용식·누락 조건·주의사항에서 다음 태그를 자동 추정합니다.

- 구름 조건 착각
- 마찰 방향/마찰 조건 오류
- 원운동 방향 오류
- 문제 조건/FBD 누락
- 운동량/에너지 보존 조건 착각
- 관성모멘트 사용 오류

## 회귀 테스트

`tests/test_beginner_strict_condition_gates.py`가 다음을 검증합니다.

- 정보 부족 입력에서 적용식 비움
- 명확한 순수 구름/미끄럼 구름은 기존처럼 처리
- 명확한 최저점/최고점 원운동과 평평한 커브는 식 제시 유지
- 비적용식 중복 제거
- 적용식 설명 패턴 보강
- 오답노트 자동 태그
