# Ambiguity Policy

## 원칙

1. 정보가 부족하면 억지로 하나의 템플릿을 확정하지 않는다.
2. 가능한 후보 유형을 제시한다.
3. 필요한 추가 조건을 명시한다.
4. 잘못 적용하면 안 되는 식을 `not_applicable_equations`에 분리한다.
5. 힘 기호나 지지 방식이 충돌하면 후보식 둘 다 제시한다.

## 예시 1: 바닥점 힘

입력: `바닥점의 힘을 구하라.`

좋은 출력:

- 수직 원운동 최저점 힘 문제 가능성.
- 줄/끈이면 `T - mg = mv²/R`.
- 트랙/레일이면 `N - mg = mv²/R`.
- 지지 방식과 반지름/속도 확인 필요.

나쁜 출력:

- 무조건 `N = mg + mv²/R` 확정.

## 예시 2: rail + tension

입력: `bead on a rail, tension at bottom`

좋은 출력:

- rail은 수직항력 N, tension은 장력 T이므로 표현 충돌.
- 트랙/레일 문제라면 `N - mg = mv²/R`.
- 줄 문제라면 `T - mg = mv²/R`.
- 문제 문장 또는 그림에서 지지 방식을 확인해야 함.

## 예시 3: 원뿔진자 후보

입력: `줄이 수직과 각도를 이루며 원운동한다.`

좋은 출력:

- 원뿔진자 가능성 있음.
- 수평 원운동 여부와 각도 기준 확인 필요.
- 원뿔진자라면 `T cosθ = mg`, `T sinθ = mω²r`, `r = L sinθ`.

## 예시 4: banked curve limiting speed

입력: `banked curve limiting speed with friction`

좋은 출력:

- 마찰 있는 경사진 커브 한계속도 후보.
- 최대속도/최소속도 또는 미끄러지려는 방향 확인 필요.
- 평평한 커브식은 적용식으로 금지.
