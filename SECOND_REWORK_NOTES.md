# 2차 개선 노트

## 목적

1차 개선안은 키워드 오진 방지와 구조 분리를 중심으로 개선되었지만, 여전히 “풀이 전략 추천”에 머무르는 한계가 있었다. 2차 개선에서는 문제 문장을 동역학 풀이에 필요한 물리 모델로 바꾸고, 문제 유형별 풀이 골격을 출력하는 방향으로 보완했다.

## 핵심 변경 사항

### 1. 문제 구조화 출력 추가

`dynamics_core/modeling.py`를 새로 추가했다. 이제 진단 결과에 다음 항목이 구조적으로 출력된다.

- 문제 유형
- 분석 대상
- 구할 값
- 주어진 물리량
- 운동 상태
- 작용하는 힘 후보
- 무시/제외 조건
- 구속조건/보조조건
- 보존/에너지 조건
- 추천 좌표계
- 사용 가능한 방법
- 위험한 방법

### 2. 풀이 골격 생성 추가

`SolutionBlueprint`를 추가해 단순 전략명 대신 다음 내용을 보여준다.

- FBD에 그릴 힘
- 좌표축 방향
- 사용할 기본 방정식
- 필요한 보조식
- 적용 조건
- 다음 풀이 단계
- 주의사항
- 답 해석 체크

### 3. 대표 유형별 골격 강화

다음 유형에 대해 문제별 안내를 강화했다.

- 경사면 F=ma
- 마찰 포함/마찰 무시 문제
- 연결된 물체/도르래
- 포물선 운동
- 원운동/법선-접선 좌표계
- 일-에너지 원리
- 충돌/운동량
- 고정축 회전
- 강체 일반 평면운동
- 순수 구름 운동
- 상대속도/상대가속도/순간중심
- 극좌표계 운동

### 4. 부정 표현 및 한국어 조사 처리 개선

추가 반례를 회귀 테스트에 포함했다.

- `마찰계수 μ가 있지만 마찰을 무시한다`
- `m=5 kg인 물체가 3 m를 이동한다`
- `탄성 충돌이 아니며`
- `충돌하지 않고`
- `공기저항을 무시하지 않는다`
- `회전하지 않고 병진운동한다`
- `kg인`, `kg의`, `kg짜리`, `N의`, `m를`, `rad/s로` 등 단위 뒤 조사 처리

### 5. 계산기 보완

포물선 계산기에 `공기저항 고려` 조건을 추가했다. 이 조건이 켜져 있으면 표준 포물선 공식 계산을 중단하고, drag 모델 또는 수치해석이 필요하다는 경고를 반환한다.

### 6. 테스트 확대

`regression_tests.py`를 2차 검수용으로 확대했다.

- 오진 방지 테스트
- 한국어 단위/조사 테스트
- 대표 동역학 유형별 전략 테스트
- 문제 구조화/풀이 골격 생성 테스트
- 계산기 경계값 테스트

현재 실행 결과:

```text
OK: all 2nd-rework regression tests passed
```

## 남은 한계

이 앱은 여전히 “완전 자동 정답 생성기”가 아니다. 그림이 핵심인 복잡한 교재 문제, 3차원 강체 운동, 비표준 기호가 많은 문제, 긴 서술형 문제는 사용자의 검토와 교재 해설 비교가 필요하다.

이번 2차 개선의 목표는 정답 자동 산출이 아니라, 학생이 문제를 보고 다음 과정을 따라가도록 돕는 것이다.

1. 분석 대상을 고른다.
2. 힘을 그린다.
3. 좌표축을 잡는다.
4. 구속조건을 세운다.
5. 적절한 원리를 고른다.
6. 기본 방정식과 보조식을 연결한다.
7. 단위와 부호를 검산한다.
