# 재작업 반영 내역

## 인정한 문제

기존 앱은 Streamlit UI와 여러 기능의 형태는 있었지만, 핵심 판단 로직이 단순 키워드 검색에 크게 의존했습니다. 특히 한국어 부분문자열과 영어 부분문자열에서 오진 위험이 있었습니다. 계산기 입력 검증, 테스트 범위, 영속 학습 기록, AI 사용 안내, 코드 구조도 납품용 기준에는 부족했습니다.

## 수정 방향

1. 문제 문장 분석을 `dynamics_core/parser.py`로 분리했습니다.
2. 풀이 전략 추천을 `dynamics_core/strategy_engine.py`로 분리했습니다.
3. 계산기와 검증을 `calculators.py`, `validators.py`로 분리했습니다.
4. FBD/오개념/빠진 요소 피드백을 `feedback.py`로 분리했습니다.
5. SQLite 학습 기록을 `storage.py`로 추가했습니다.
6. UI는 `app.py`에 남기되 핵심 로직은 모듈에서 호출하도록 바꿨습니다.
7. `regression_tests.py`에 오진 방지 테스트와 대표 문제 테스트, 계산기 경계값 테스트를 추가했습니다.

## 현재 테스트 통과 항목

- 매끈한 ≠ 끈
- 구하라 ≠ 구형 물체
- momentum ≠ moment
- 마찰이 없는 ≠ 마찰 있음
- 순간중심 ≠ 충격량
- 3 m ≠ 질량 m
- 포물선 운동학
- F=ma 경사면
- 마찰 없음 에너지
- 에너지 + 원운동 복합
- 충돌 후 속도
- 순수 구름 복합
- 고정축 강체 회전
- 질량 있는 도르래 복합
- 곡률반지름 원운동
- 상대속도 문제
- 계산기 음수/0/범위 검증

# 2차 개선 반영 내역

사용자의 2차 검토 의견에서 지적한 핵심 문제는 “전략 추천에 머물고, 실제 동역학 풀이 골격을 충분히 만들지 못한다”는 점이었다. 이를 반영해 다음을 추가했다.

1. `dynamics_core/modeling.py` 추가
   - 문제 구조화 담당
   - 분석 대상, 힘, 무시 조건, 좌표계, 구속조건, 위험한 방법 등을 생성

2. `ProblemModel`, `SolutionBlueprint` 데이터 모델 추가
   - 단순 추천 결과와 별개로 물리 모델과 풀이 골격을 저장

3. 앱 화면에 “문제 구조화”와 “문제별 풀이 골격” 섹션 추가
   - FBD 힘 목록
   - 좌표축 방향
   - 기본 방정식
   - 보조식
   - 적용 조건
   - 다음 풀이 단계

4. 파서 반례 처리 강화
   - 마찰계수는 있지만 마찰 무시
   - 공기저항 무시/무시하지 않음 구분
   - 충돌함/충돌하지 않음 구분
   - 탄성 충돌/스프링 탄성 구분
   - 회전함/회전하지 않음 구분
   - 한국어 조사 붙은 단위 처리

5. 회귀 테스트 확대
   - 오진 방지
   - 단위/조사 처리
   - 대표 유형별 추천
   - 구조화/풀이 골격
   - 계산기 경계값 검증

6. 계산기 보완
   - 포물선 계산기에 공기저항 고려 여부 추가
   - 공기저항을 고려하면 표준 포물선 공식 계산을 중단하고 경고

이번 버전도 완전 자동 풀이기는 아니며, 문서에 지원 범위와 한계를 더 명확히 적었다.
